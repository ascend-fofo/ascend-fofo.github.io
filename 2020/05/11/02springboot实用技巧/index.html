从配置文件中获取配置

疑问为什么要从配置文件中获取配置？

1 可以当做常量来使用

如何从配置文件中获取配置？

自己查idea格式化快捷键

坑：

1中文乱码

2不要将属性写为user.name 这样会获取你电脑的用户名

获取方式一

value("${}")
//细节 注意配置中的字符串会默认加上""符号 所以不需要加

课外 # 与$符号的区别？

做完练习后我们想一下 这个有什么缺点？

有点麻烦，如果要获取若干配置的话，那么就需要一个个取值

获取方式二

@ConfigurationProperties(prefix = "这里写头")
@componet //一定要记得加

这里返回空json，推荐安装谷歌浏览器json插件（json-handle）

优势

可以大范围多字段一次性获取多个属性

使用配置文件创建多环境以及环境的选择

dev 表示开发环境

sit 表示测试环境

prd 表示正式线上环境

痛点，不同的环境下的数据库链接不同，用户名，密码等等配置都有可能不同

如果换个环境就去该配置文件的话太麻烦了

凡是没有使用多环境的公司，都是垃圾

所以产生了多环境的配置文件与选择配置文件

步骤：

①创建多环境配置规范

例如：application-dev.properties

要在原先的配置环境下新加一个配置文件，然后在application后加-符号加上你需要的环境名即可

②选择某个环境

spring.profiles.active=dev  //后面这里写的是环境名（也就是-后面的字符）
