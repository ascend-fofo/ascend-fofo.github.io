Swagger

接口文档

接口文档是给谁看的？

给前端开发看的

为什么要写接口文档？

就是可以让前后端进行对接

课外

@GetMapping(value = "/getGoods/{str}")
public Goods getGoods(@PathVariable(name = "str") String str)

@Validated

步骤

①依赖

<!--swagger 开始-->
<dependency>
   <groupId>io.springfox</groupId>
   <artifactId>springfox-swagger2</artifactId>
   <version>2.9.2</version>
</dependency>
<!--swagger 结束-->
<!-- swaggerui开始 -->
       <dependency>
           <groupId>com.github.xiaoymin</groupId>
           <artifactId>swagger-bootstrap-ui</artifactId>
           <version>1.9.3</version>
       </dependency>
       <!-- swaggerui结束 这里相当于给了一层类似于bootstrap的ui界面-->

②配置

@Configuration
@EnableSwagger2
@EnableSwaggerBootstrapUI
public class SwaggerConfiguration {
}

swagger-bootstrap-ui的访问地址

{host}:{port}doc.html

swagger中常用注解**

@Api(tags = "实时查询客户各种操作(例如登录，拓客等)的流水日志") //也可以用来表示版本号
@ApiOperation("获取商品列表") //描述接口的功能

@ApiModel(value = "商品实体类") //描述实体类的
@ApiModelProperty(value="描述字段" ，required=true) //这里注意这里的required默认是false
